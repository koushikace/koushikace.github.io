Online Payment Fraud Detection

This project aims to build and evaluate machine learning models for detecting fraudulent transactions in an online payment dataset. With the increasing volume of digital transactions, identifying and preventing fraud is crucial for financial institutions and e-commerce platforms.

Dataset & Features:

The core dataset is `new_data.csv`, simulated financial transaction data from a mobile money service. It includes features like `step`, `type` (CASH_OUT, PAYMENT, DEBIT, TRANSFER, CASH_IN), `amount`, `nameOrig`, `oldbalanceOrg`, `newbalanceOrg`, `nameDest`, `oldbalanceDest`, `newbalanceDest`, `isFraud` (target variable), and `isFlaggedFraud`. (Note: Dataset size is approx. 470 MB, managed with Git LFS).

Data Exploration & Preprocessing:

Detailed in `onilepayment.ipynb` notebook. Steps include: Initial Data Loading, Categorical/Numerical Feature Identification, EDA (Transaction Type, Fraud Distribution, Time Step, Correlation Heatmap), Feature Engineering (One-Hot Encoding for `type`), Feature/Target Separation, and Data Splitting (70/30 train/test split, `random_state=42`).

Modeling & Evaluation:

Evaluated classification models: Logistic Regression, XGBoost Classifier, Random Forest Classifier. Performance assessed using ROC AUC Score, critical for imbalanced datasets. Confusion Matrix visualized for XGBoost Classifier.

Key Libraries:

NumPy, Pandas, Matplotlib, Seaborn, scikit-learn, XGBoost.

Performance Analysis of NLP Techniques on Laptop Reviews

This project focuses on performing sentiment analysis on Amazon laptop reviews, classifying them as either "positive" or "negative/neutral" based on text content. It evaluates various NLP techniques and machine learning models.

Dataset:

`Cleansed_laptop data.csv`, containing `comment` (text) and `rating` (1-5 stars). Ratings 1-3 are mapped to 0 (Negative/Neutral), 4-5 to 1 (Positive).

Key Features & Techniques:

Data Loading & Initial Exploration. Text Preprocessing: Lowercasing, Punctuation/Special Character Removal, Stop Word Removal, Lemmatization (NLTK WordNetLemmatizer). Feature Extraction: TF-IDF. Handling Class Imbalance: SMOTE (Synthetic Minority Over-sampling Technique).

Model Training & Evaluation:

Data Splitting (training, validation, testing). Initial model: Decision Tree Classifier. Performance Metrics: Confusion Matrix, Precision, Recall, Accuracy, F1-Score. Achieved 80.43% across all metrics on the validation set, attributed to effective SMOTE handling.

Key Libraries:

Pandas, scikit-learn, imblearn, nltk, regex.

Default of Credit Card Clients Prediction

This project involves building models to predict whether a credit card client will default on their payment in the next month. The dataset contains rich demographic and financial information.

Dataset Features:

`ID`, `LIMIT_BAL` (credit amount), `SEX`, `EDUCATION`, `MARRIAGE`, `AGE`, `PAY_0 - PAY_6` (payment status), `BILL_AMT1 - BILL_AMT6` (bill statements), `PAY_AMT1 - PAY_AMT6` (previous payments), and `default payment next month` (target: 1 = yes, 0 = no).

Types of Models Explored:

  • Binary Classification: Predicting default (yes/no) using Logistic Regression, Decision Trees, Random Forests, Neural Networks.
  • Multi-class Classification: Predicting probability of default with more than two classes, using Multinomial Logistic Regression or SVM.
  • Regression Model: Estimating default probability as a continuous variable using Linear Regression, Decision Trees, Random Forests, Neural Networks.
  • Clustering Model: Grouping clients based on demographic/financial info using K-means or Hierarchical Clustering.
  • Association Rule Mining Model: Identifying patterns (e.g., variables associated with default) using Apriori or FP-growth.
